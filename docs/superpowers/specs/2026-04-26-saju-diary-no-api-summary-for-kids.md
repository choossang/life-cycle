# 사주 + 4행일기 시스템 한눈에 보기 (쉬운 설명)

작성일: 2026-04-26

## 1. 이 시스템은 뭐예요?

이건 **점수판 + 일기장**이에요.

- 점수판: 사주 점수(job/gui/hlt)
- 일기장: 4행일기(사실/감정/배움/선언)

중요!
- 점수판은 **그대로** 두고
- 일기장은 **설명 메모**로만 써요.

---

## 2. 왜 API를 안 써요?

보안과 합법적인 사용을 더 중요하게 보기 때문이에요.

그래서 이렇게 해요:
- 화면 보기: 서버가 직접 만들어서 보여줌(SSR)
- 저장/삭제: 버튼(form POST)으로만 보냄
- 브라우저에서 `fetch('/api/...')` 같은 호출은 안 씀

---

## 3. 버튼을 누르면 어떻게 돼요?

### 저장 버튼
1) 내가 4행일기를 씀
2) 저장 버튼 클릭
3) 서버가 검사하고 저장
4) 다시 화면으로 돌아옴

### 삭제 버튼
1) 삭제 버튼 클릭
2) 서버가 검사하고 삭제
3) 다시 화면으로 돌아옴

---

## 4. 특정 달을 보면 뭐가 나오나요?

기준 달을 고르면 4가지 Top5를 보여줘요.

1) 과거 기회 Top5
2) 과거 위기 Top5
3) 미래예측 기회 Top5
4) 미래예측 위기 Top5

각 줄에는 이것이 같이 보여요:
- 월
- 강도 점수
- 신뢰도(%)
- 왜 그렇게 나왔는지(근거 요약)

---

## 5. 과거와 미래는 뭐가 달라요?

- 과거 Top5: 실제 기록(일기/이벤트) 중심
- 미래 Top5: 예측 중심

그래서 미래는 꼭 **신뢰도(%)**를 같이 보여줘요.

---

## 6. 안전하게 쓰기 위한 약속

1. 점수는 일기로 바꾸지 않기
2. 저장/삭제는 form POST만 사용
3. CSRF 토큰으로 "진짜 요청"인지 확인
4. 사용자 글은 안전하게 표시(XSS 차단)
5. 비밀번호/시크릿 키는 Render 환경변수로만 관리

---

## 7. 한 줄 결론

**점수는 고정, 일기는 근거, 저장은 버튼, 보안은 최우선.**
